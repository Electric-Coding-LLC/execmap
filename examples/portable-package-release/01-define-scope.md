# Define Scope

[Back to Execution Map](./EXECMAP.md)

## Goal

Define the smallest useful portability initiative so implementation stays
focused on install and launch reliability.

## Tasks

- Identify the practical adoption problem the initiative is solving.
- Define what stays inside the initiative.
- Define what stays outside the initiative.

## Constraints

- Keep the initiative narrow enough to finish cleanly.
- Do not broaden the work into a general product redesign.

## Exit Criteria

- The initiative can be described clearly in one sentence.
- The primary scope and non-goals are explicit enough to guide later steps.

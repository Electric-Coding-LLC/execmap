# `execmap` `0.1`

## Scope

`0.1` ships the existing bounded-planning workflow:

- `init` scaffolds a new initiative folder with `EXECMAP.md` and numbered step
  docs
- `next` prints the next unchecked execution-map item
- `check` validates required sections, linked step docs, and backlinks
- `skills/execmap/` stays aligned with the same workflow for Codex use

## Install Contract

`0.1` is a Bun-first npm package. Installed use requires Bun `1.3+`.

Recommended paths:

```bash
bunx @electric_coding/execmap init "portable package release"
```

```bash
bun install -g @electric_coding/execmap
execmap check plans/portable-package-release
```

In a repo checkout, the equivalent path is:

```bash
bun run src/cli.ts check examples/portable-package-release
```

## Verification

Release verification for `0.1` includes:

- `bun run typecheck`
- `bun test`
- `bun run check:example`
- `bun run check:packed-install`

The packed-install smoke test creates a tarball, installs it into a clean temp
directory with Bun, and runs the installed `execmap` binary against a fresh
scaffold.

## Known Limits

- `0.1` does not provide a Node-only runtime path
- `0.1` keeps the CLI intentionally small; there are no editing or migration
  subcommands yet
- release publication itself is separate from repo-local verification

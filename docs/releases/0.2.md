# `execmap` `0.2`

## Scope

`0.2` adds the first supported mutating CLI workflow:

- `done` marks the first unchecked execution-map item complete
- `done` resolves the target through the same repo-root, `PLAN.md`, and direct
  execmap paths as `next` and `check`
- the command rewrites only the targeted checkbox line so hand-authored
  markdown structure stays intact

`0.2` keeps the release intentionally small. The public CLI surface becomes:

- `init`
- `next`
- `done`
- `check`

## Install Contract

`0.2` stays a Bun-first npm package. Installed use requires Bun `1.3+`.

Recommended paths:

```bash
cd my-repo
bunx @electric_coding/execmap done
```

```bash
bun install -g @electric_coding/execmap
cd my-repo
execmap done
```

In a repo checkout, the equivalent path is:

```bash
bun run src/cli.ts done .
```

## Verification

Release verification for `0.2` includes:

- `bun run typecheck`
- `bun test`
- `bun run check:example`
- `bun run check:packed-install`

The CLI tests cover the new `done` flow from the repo root, verify that
`next` advances after a successful `done`, and verify the clear failure path
when no unchecked steps remain.

## Known Limits

- `0.2` does not mutate `PLAN.md` or switch the active initiative
- `0.2` does not reopen completed steps
- `0.2` does not insert, delete, or renumber execution-map steps
